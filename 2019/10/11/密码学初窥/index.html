密码学初窥

Base64 MD5 HMAC AES RSA 指纹识别
发布日期:   2019-10-11

Base64

base64是一种完全可反编译的编码方式，因为其编码算法完全公开，所以分分钟会被破解，所以这个一定不能用于密码的‘加密’。一些不需要特别加密的信息，比如用户名，我们可以用base64进行编码，让人一眼看不出来是什么就可以了。

base64是网络上最常见的用于传输8bit字节代码的编码方式之一，base64要求把每3个8bit字节转换为4个6bit的字节，然后把6bit再添加两个高位0，组成4个8bit的字节，也就是转换后字符串要变原来长1/3。

比如说：

转换前 aaaaaabb ccccdddd eeffffff
转换后 00aaaaaa 00bbcccc 00ddddee 00ffffff

上面的三个字节是原文，下面四个字节是转换后的base64编码，其前两位均为0.

转换后，我们用一个码表来得到我们想要的字符串（最终的base64编码）

我们再看一个例子：

转换前 10101101 10111010 01110110
转换后 00101011 00011011 00101001 00110110

对应的十进制是 43 27 41 54
对应码表中的 rbp2

可是原文的字节数应该是3的倍数把，如果这个条件不满足的话，应该怎么办呢？

解决方案是：原文的字节不够的地方可以用0来补足，转换成base64编码用=来代替。这就是为什么有的base64编码会以一个或者两个等号结束的原因，但等号最多只有两个。

因为：

余数 = 原文字节数  MOD 3

所以余数在任何情况下都只可能是0 1 2中的一个。如果余数是0的话，表示原文的字节数正好是3的倍数。如果是1的话，就让base64编码是3的倍数，就要补2个等号；同理，如果是2的话，就要补1个等号。

因为=字符并不在base64码表里，其意义在于结束符号，在base64编码时遇见=即可知道base64编码字符串结束了。

MD5

全称是Message Digest Algorithm 5，翻译为 消息摘要算法第五版。它能对输入信息生成唯一的128位（16字节）32个字符的散列值。输入两个不同的明文不会得到相同输出值。

其过程是不可逆的

由于MD5加密算法具有较好的安全性，而且免费，因此该算法被广泛使用。

主要用于数字签名，文件完整性验证以及口令加密等方面。

当用户注册，登录时，我们可以直接对密码进行md5操作，用户登录时，系统把用户输入的密码计算md5值，然后再去和系统中保存的md5进行比较，只要一样就可以登录，系统并不知道用户的密码是什么。

MD5的特点：

  • 压缩性：任意长度的数据，计算出来的md5值长度都是固定的
  • 容易计算：从原始数据计算出md5的值非常容易
  • 抗修改性：对原始数据的任何改动，哪怕只修改一个字节，所得到的的md5的散列值都会有很大区别
  • 弱抗碰撞：已知原始数据和其md5值，想找到一具有相同md5值的数据是非常困难的（伪造数据）
  • 强抗碰撞：想到两个不同的数据，使他们具有相同的md5散列值是非常困难的

不过现在，MD5已不再绝对安全，网上有很多可以查询MD5原始数据的网站。

针对这种情况，需要对md5稍作修改，增加解密的难度。

  • 加盐 salt： 在明文的固定位置插入随机串，然后再进行md5
  • 先加密，后乱序：先对明文进行md5，然后对加密得到的md5字符串进行乱序。

宗旨，就算被人攻破数据库，也无法正确解密正确的明文。

AES

高级加密标准advanced encryption standard，简称AES。是美国联邦政府采用的一种区块加密标准。AES设计有三个密钥长度：128，192，256位，相对而言，AES的128密钥比DES的56密钥强1021倍。AES算法主要包括三个方面：轮变化，圈数和密钥扩展。

AES加密的特点：

  • 安全性强，高性能，高效率，易用灵活
  • 在软件和硬件上都能快速的加密解密，只需要很少的存储资源

AES的使用：除了类库本身只有Config类里面包含AESCrpt.h，只有两个方法用到。一个是保存用户名和密码，密码使用了AES加密，另一个是解密密码后再返回这个密码；保存用户名和密码是将用户名和密码放到了本地的一个沙盒之中，获取的时候直接从本地读取加密后的文件，经过解密和服务器上用户数据进行比较。

-(void)saveUserNameAndPwd:(NSString *)userName andPwd:(NSString *)pwd  
{  
    NSUserDefaults * settings = [NSUserDefaults standardUserDefaults];  
    [settings removeObjectForKey:@"UserName"];  
    [settings removeObjectForKey:@"Password"];  
    [settings setObject:userName forKey:@"UserName"];  

    pwd = [AESCrypt encrypt:pwd password:@"pwd"];  

    [settings setObject:pwd forKey:@"Password"];  
    [settings synchronize];  
}  
-(NSString *)getPwd  
{  
    NSUserDefaults * settings = [NSUserDefaults standardUserDefaults];  
    NSString * temp = [settings objectForKey:@"Password"];  
    return [AESCrypt decrypt:temp password:@"pwd"];  
}

正如官方给出示例用法一样，AES的使用非常简单，首先要添加头文件 #import “AESCrypt.h”，使用示例如下：

NSString *pwdKey = @"新风作浪";  
NSString *password = @"duxinfeng123456";  

NSString *encryptedPWD = [AESCrypt encrypt:password password:pwdKey];  
NSString *decryptedPWD = [AESCrypt decrypt:encryptedPWD password:pwdKey];  

NSLog(@"加密后密码:%@  解密后密码: %@",encryptedPWD,decryptedPWD);

RSA

RSA是非对称加密算法，它需要两个密钥：

  • 公钥（public key）
  • 私钥 （private key）

公钥和私钥是一对，如果用公钥对数据加密，只有对应的私钥才能解密；如果用私钥对数据进行加密，那么只有对应的公钥才能解密。

优点：算法复杂度强，安全性依赖于算法于密钥
缺点：加密速度没有对称加密速度快，一般用于小数据
